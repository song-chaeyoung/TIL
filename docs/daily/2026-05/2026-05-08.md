### 응용 프로그램 보안 설정 개념

- 응용 프로그램 보안 설정은 프로그램이나 시스템이 공격에 악용되지 않도록 권한, 접근, 정보 노출, 실행 환경을 제한하는 설정

**실행 프로세스 권한 설정**

- 프로그램이 불필요하게 관리자 권한으로 실행되지 않도록 제한하는 것
- 최소 권한 원칙과 관련 있음
  **운영체제 접근 제한**
- 사용자, 파일, 디렉터리, 서비스에 대한 접근 권한을 제한하는 것
- 인가되지 않은 사용자가 시스템 자원에 접근하지 못하게 함
  **정보 수집 제한**
- 운영체제 버전, 서비스 정보, 계정 정보, 환경 설정 정보 등이 외부에 노출되지 않도록 제한
- 공격자가 취약점 분석에 활용할 수 있는 정보를 줄이는 목적
  **물리적 보안**
- 서버실 출입 통제, 장비 잠금, CCTV, 출입 기록 관리 등
- 응용 프로그램 설정과는 다른 영역이지만 전체 보안에서는 중요함

### TCP 세그먼트 구조

송신지 포트, 수신지 포트

- 송수신하는 포트 번호

**순서 번호 (sequence number)**

- 순서 번호가 명시되는 필드
- 순서 번호
  - 송수신되는 세그먼트 데이터의 첫 바이트에 부여되는 번호
  - 세그먼트의 올바른 송수신 순서를 보장하기 위한 번호
    순서 번호 = 초기 순서번호 + 송신산 바이트 수
  - 초기 순서 번호 (ISN, Initial Sequence Number)는 무작위 값
  - 세그먼트 B의 순서번호는 초기 순서 번호인 100에서 500바이트 떨어진 셈이므로 600

**확인 응답 번호 (acknowledgment number)**

- 상대 호스트가 보낸 세그먼트에 대한 응답
- 다음으로 수신하기를 기대하는 순서 번호가 명시
- 수신지가 다음으로 받기를 기대하는 순서 번호 (일반적으로 수신한 순서 번호 + 1)
- 확인 응답 번호 값을 보내기 위해서는 제어 비트에서 승인을 나타내는 비트인 **ACK 플래그**를 1로 설정

**제어 필드 (control bits) / 플래그 비트 (flag bits)**

- 현재 세그먼트에 대한 부가 정보
- 기본적으로 8비트로 구성
  - ACK : 세그먼트의 승인을 나타내기 위한 비트
  - SYN : 연결을 수립하기 위한 비트
  - FIN : 연결을 종료하기 위한 비트

**윈도우 (window)**

- 수신 윈도우(한 번에 수신하고자 하는 데이터의 양)의 크기가 명시

### 연결 수립 : 쓰리 웨이 핸드셰이크

- 쓰리 웨이 핸드셰이크(three-way handshake) 세 개의 단계로 이루어진 TCP의 연결 수립 과정

### 연결 종료 : 포 웨이 핸드셰이크

- 송수신 호스트가 각자 한 번씩 FIN과 ACK를 주고 받으며 TCP가 연결 종료
  - 액티브 클로즈 (active close) : 종료 시작 호스트의 종료 과정
  - 패시브 클로즈 (passive close) : 종료 수락 호스트의 종료 과정
