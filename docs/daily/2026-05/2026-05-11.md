### 간단 요약

1. IP는 비신뢰성, 비연결형 프로토콜이다.
   → 도착 보장, 순서 보장, 재전송을 직접 하지 않는다.
2. TCP와 UDP는 전송 계층 프로토콜이다.
   → 포트 번호로 애플리케이션을 식별한다.
3. TCP는 연결형, 신뢰성 있는 통신을 제공한다.
   → 3-way handshake로 연결을 수립하고,
   4-way handshake로 연결을 종료한다.
4. UDP는 비연결형, 비신뢰성 기반의 빠른 프로토콜이다.
   → 실시간성이 중요한 상황에2서 유리하다.
5. 포트 번호는 호스트 안의 애플리케이션을 식별한다.
   → IP 주소는 호스트, 포트는 프로세스 구분.
6. NAT/NAPT는 사설 IP와 공인 IP 사이의 주소 변환 기술이다.
   → NAPT는 포트 번호까지 이용해 하나의 공인 IP를 여러 내부 호스트가 공유하게 한다.
7. ICMP는 IP 패킷 전송 과정의 오류 보고와 진단 정보를 제공한다.
   → ping 같은 명령에서 사용된다.
8. TCP 오류 제어는 ACK, 타임아웃, 재전송을 이용한다.
   → Stop-and-Wait, Go-Back-N, Selective Repeat 같은 방식이 있다.
9. TCP 흐름 제어는 수신자의 처리 능력을 고려한다.
   → 수신 윈도우와 슬라이딩 윈도우를 사용한다.
10. TCP 혼잡 제어는 네트워크 혼잡 정도를 고려한다.
    → 혼잡 윈도우, AIMD, Slow Start, Congestion Avoidance,
    Fast Retransmit, Fast Recovery를 사용한다.
11. ECN은 패킷 손실 전에 혼잡을 명시적으로 알려준다.
    → 라우터가 혼잡 표시를 남기고, 수신자가 ACK로 송신자에게 알린다.
