## 배운 내용

### RDBMS — 키

- 포함 관계가 핵심: **슈퍼 키 ⊃ 후보 키 ⊃ 기본 키**
  - 슈퍼 키: 유일성만 만족 (불필요한 필드 포함 가능)
  - 후보 키: 유일성 + **최소성** 만족하는 최소 집합
  - 기본 키(PK): 후보 키 중 대표 1개. 중복 금지 + **NULL 금지**
- 대체 키 = PK가 되지 못한 후보 키 / 외래 키(FK) = 다른 테이블 PK 참조

### RDBMS — 무결성 제약 4종

- 도메인(타입·범위) / 키(중복 금지) / 엔티티(PK 고유+NULL 불가) / 참조(FK는 참조 PK 값 또는 NULL)
- 참조 대상 삭제 시 동작 4가지: restrict / set default / set null / **cascade**

### 자료구조 복습에서 다시 확인한 것

- 스택/큐 연산 추적은 정확히 풀림 (LIFO/FIFO 감각 OK)
- **오해했던 것**: "탐색은 무조건 해시가 답"이 아니다 — 해시 O(1)은 버킷 메모리를 대가로 치름. 정렬된 데이터 + 메모리 제약이면 배열 이진 탐색 O(log n)이 정답
- 약점 유형: 시나리오 보고 자료구조 선택 + 복잡도 비교 → 2차 복습(6/16) 때 재확인
