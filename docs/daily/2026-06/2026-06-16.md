## 배운 내용

### NoSQL 유형

- **키-값 (Redis, Memcached)**: 가장 단순한 구조. 인 메모리 저장으로 빠른 접근. 캐시·세션 용도, 다른 DB의 보조로 쓰이는 경우 많음
- **도큐먼트 (MongoDB)**: 레코드를 JSON 형태의 도큐먼트 단위로 저장. 고정 스키마 없어 유연함. 도큐먼트 → 컬렉션 → DB 구조. 내부적으로 BSON 사용
- **칼럼 패밀리 (Cassandra, HBase)**: 행·열 개념은 있지만 테이블 구조가 엄격하지 않음. 로우 키로 행 식별. 열을 동적으로 추가 가능

### RDBMS vs NoSQL

| | RDBMS | NoSQL |
|---|---|---|
| 스키마 | 고정 | 유연 |
| 스케일 아웃 | 어려움 | 특화 |
| ACID | 엄격 준수 | 다소 희생 |
| 무결성·일관성 | 높음 | 낮음 |
| 적합 환경 | 정형 데이터, 무결성 중요 | 대용량, 분산, 비정형 |

### 헷갈렸던 것

- MongoDB의 도큐먼트가 RDBMS의 **행(row)**에 대응한다는 점 (테이블이 아님)
- NoSQL이 성능을 얻는 대신 희생하는 게 ACID와 정규화 — 무결성 중요한 환경에선 RDBMS가 맞음
